# GoKartARLine 使用教程

本教程面向第一次使用 `GoKartARLine` 的用户，说明如何从 GitHub 构建未签名 IPA、如何在 Xcode 真机运行、如何导入 AI 生成的赛道数据，以及如何在卡丁车场安全使用 AR 行车线。

## 1. App 是做什么的

`GoKartARLine` 是一个纯 iPhone 卡丁车 AR 动态行车线 App：

1. 使用外部 AI 根据赛道俯视图预先生成行车线 JSON 数据。
2. App 导入 JSON 或 GPX 文件。
3. App 使用 iPhone 的 GPS、IMU 和 ARKit，把行车线叠加到实时摄像头画面上。
4. 行车线颜色表示驾驶提示：绿色全油门、橙色松油、红色刹车。

重要：这个 App 不需要你先跑最快圈，也不依赖外部硬件。

## 2. 准备工作

### 2.1 设备要求

- iPhone 真机
- iOS 17.0 或更高版本
- 支持 ARKit 的设备
- 室外或 GPS 信号较好的卡丁车场

### 2.2 开发/构建要求

如果要自己编译运行，需要：

- macOS
- Xcode 15.4 或更高版本
- Apple ID
- 如需安装到真机，需要配置 Signing Team

Windows 不能原生安装 Xcode，也不能直接编译 iOS App。Windows 可以写代码、推送 GitHub，然后使用 GitHub Actions 的 macOS runner 构建。

## 3. 从 GitHub Actions 构建未签名 IPA

仓库已配置 GitHub Actions：`Build Unsigned IPA`。

### 3.1 自动构建

每次推送到 `main` 分支后，GitHub 会自动触发构建。

### 3.2 手动构建

1. 打开 GitHub 仓库页面。
2. 点击顶部 `Actions`。
3. 选择左侧 `Build Unsigned IPA`。
4. 点击右侧 `Run workflow`。
5. 分支选择 `main`。
6. 点击绿色的 `Run workflow` 按钮。

### 3.3 下载构建产物

构建成功后：

1. 打开对应 workflow run。
2. 滚动到页面底部 `Artifacts` 区域。
3. 下载 `GoKartARLine-unsigned-ipa`。
4. 解压后可看到 `GoKartARLine-unsigned.ipa`。

### 3.4 未签名 IPA 的限制

该 IPA 是通过 `CODE_SIGNING_ALLOWED=NO` 构建的未签名产物：

- 不能直接安装到普通 iPhone。
- 适合做编译验证、代码审查、后续再签名。
- 如果要真机安装，需要使用 Apple Developer 签名、描述文件或合法再签名流程。

## 4. 使用 Xcode 真机运行

这是最推荐的测试方式。

1. 在 Mac 上克隆或下载仓库。
2. 用 Xcode 打开 `GoKartARLine.xcodeproj`。
3. 选择项目 target：`GoKartARLine`。
4. 打开 `Signing & Capabilities`。
5. 设置自己的 `Team`。
6. 如有需要，修改 `Bundle Identifier`，例如改为 `com.yourname.GoKartARLine`。
7. 连接 iPhone 真机。
8. 在 Xcode 顶部选择该 iPhone。
9. 点击 Run。
10. 首次启动时允许相机、定位、运动传感器、相册写入权限。

如果 iPhone 提示不信任开发者账号，需要到：

`设置` → `通用` → `VPN与设备管理` → 信任你的开发者证书。

## 5. 赛道数据格式

App 支持导入 JSON 和 GPX。

推荐使用 JSON，格式如下：

```json
{
  "trackName": "XX卡丁车场",
  "trackLength": 1200,
  "cornerCount": 8,
  "points": [
    {
      "latitude": 31.234567,
      "longitude": 121.345678,
      "speed": 65,
      "color": "green"
    }
  ]
}
```

字段说明：

- `trackName`：赛道名称。
- `trackLength`：赛道长度，单位米。
- `cornerCount`：弯道数量。
- `points`：行车线轨迹点数组，至少 100 个点。
- `latitude`：WGS84 纬度。
- `longitude`：WGS84 经度。
- `speed`：建议速度，可用于展示或后续分析。
- `color`：驾驶提示颜色，只能是 `green`、`orange`、`red`。

颜色含义：

- `green`：全油门。
- `orange`：松油。
- `red`：刹车。

仓库根目录提供了测试文件：`SampleTrack.json`。

## 6. 如何生成 AI 赛道 JSON

你可以把赛道俯视图、卫星图或手绘赛道图发给 Claude/GPT，让它生成 JSON。

建议提示词：

```text
请根据这张卡丁车赛道俯视图生成行车线 JSON。
要求：
1. 输出 WGS84 经纬度轨迹点。
2. 至少 100 个点。
3. 每个点包含 latitude、longitude、speed、color。
4. color 只能是 green、orange、red。
5. green 表示全油门，orange 表示松油，red 表示刹车。
6. JSON 顶层包含 trackName、trackLength、cornerCount、points。
7. 只输出 JSON，不要输出解释文字。
```

如果只有图片没有真实地理坐标，AI 可能无法生成准确 WGS84 经纬度。此时需要提供：

- 赛道起点经纬度。
- 赛道方向。
- 赛道大概尺寸。
- 或者一份 GPX/地图参考数据。

## 7. 导入赛道数据

### 7.1 通过 App 内按钮导入

1. 启动 App。
2. 点击底部 `导入`。
3. 在文件选择器中选择 `.json` 或 `.gpx` 文件。
4. 导入成功后，App 会自动选择该赛道。

### 7.2 通过 AirDrop 或文件 App 导入

1. 把 JSON 文件通过 AirDrop 发送到 iPhone。
2. 保存到 `文件` App。
3. 在 GoKartARLine 中点击 `导入`。
4. 选择该 JSON 文件。

### 7.3 管理赛道

点击底部 `赛道`：

- 点击某条赛道会先打开地图校准页；确认当前所在点和车头方向后再加载该赛道。
- 左滑可重命名。
- 左滑删除可移除本地赛道。

## 8. 校准位置

为了让 AR 行车线对齐真实赛道，需要校准。

### 8.1 自动起点校准

App 会把 JSON 的第一个轨迹点视为起点。

当车辆第一次经过起点附近约 5 米范围内时，App 会自动完成校准。

建议：

1. 先导入赛道。
2. 把车开到起点附近。
3. 低速通过起点。
4. 等待顶部或状态提示显示校准完成。

### 8.2 每圈自动修正

每次再次经过起点附近时，App 会自动修正累计误差。

这有助于抵消 GPS 漂移和 IMU 长时间积分误差。

### 8.3 选择赛道时地图校准

点击底部 `赛道` 并选择某条赛道后，App 会显示赛道俯视地图：

1. 在地图上点击你当前所在的赛道位置。
2. 拖动红色方向箭头，或使用方向滑块设置当前车头方向。
3. 点击 `加载赛道并应用校准`。
4. App 会把所选位置设为 AR 原点，并按你设置的方向旋转行车线。

这种方式适合 AI 生成赛道或没有可靠起点 GPS 的场景；正式驾驶前仍建议低速检查行车线是否贴合真实路面。

### 8.4 手动校准

如果自动校准效果不好，可以点击底部 `校准`。

手动校准逻辑：

1. 如果当前有 GPS 位置，使用当前位置作为 AR 原点。
2. 如果没有 GPS，但赛道有起点，使用赛道起点。

## 9. 上车使用步骤

推荐流程：

1. 在开阔区域打开 App。
2. 确认定位权限、相机权限、运动权限已允许。
3. 导入正确赛道 JSON。
4. 点击 `赛道`，选择当前赛道，并在地图校准页点选当前位置和车头方向。
5. 将 iPhone 横屏固定在安全位置。
6. 低速驶向起点，让自动校准进一步修正。
7. 观察 AR 行车线是否贴近地面且方向正确。
8. 根据绿色、橙色、红色提示辅助驾驶。
9. 如偏移明显，点击 `校准` 或重新经过起点。

## 10. 屏幕信息说明

顶部主提示：

- `全油门`：当前附近行车线为绿色。
- `松油`：当前附近行车线为橙色。
- `刹车`：当前附近行车线为红色。
- `请导入赛道`：当前没有可用赛道。

数据卡片：

- `车速`：GPS + IMU 融合后的估算车速。
- `刹车点`：距离最近红色刹车点的大致距离。
- `GPS`：当前定位水平精度，例如 `±5m`。

底部按钮：

- `导入`：导入 JSON/GPX。
- `赛道`：查看、选择、删除、重命名赛道。
- `校准`：手动校准位置。
- `录屏/停止`：使用 ReplayKit 录制屏幕。
- `截图`：保存当前 AR 画面到相册。
- `设置`：调整样式和性能选项。

## 11. 设置说明

点击底部 `设置`。

### 11.1 行车线样式

- `透明度`：控制行车线透明程度。
- `宽度`：默认 0.5 米，可根据视觉效果调整。
- `亮度`：控制发光强度。
- `禁用深度测试`：开启后行车线更容易显示，帧率更高，但遮挡关系不真实。

### 11.2 传感器与性能

- `GPS精度`：建议使用导航级。
- `省电模式`：开启后 AR 降到 30fps，降低耗电和发热。

### 11.3 单位

- `公制单位`：开启显示 km/h 和米。
- 关闭后显示 mph。

## 12. 常见问题

### 12.1 AR 行车线明显偏移

可能原因：

- JSON 经纬度不准。
- GPS 精度太差。
- 起点点位不准。
- 手机固定位置改变。
- ARKit 跟踪状态不佳。

解决方法：

1. 到开阔处等待 GPS 精度变好。
2. 低速重新经过起点。
3. 点击 `校准`。
4. 检查 JSON 第一个点是否真的是起点。
5. 重新生成更准确的赛道数据。

### 12.2 无法导入 JSON

检查：

- `points` 是否至少 100 个。
- `color` 是否只用了 `green`、`orange`、`red`。
- 经纬度是否在合法范围。
- JSON 是否有多余注释。标准 JSON 不允许 `//` 注释。

### 12.3 GitHub Actions 构建成功但 IPA 不能安装

这是正常的。当前 IPA 是未签名产物，只用于验证编译成功。

如果要安装到真机，需要签名。

### 12.4 iPhone 上没有摄像头画面

检查：

- 是否是真机，不是模拟器。
- 是否允许相机权限。
- 设备是否支持 ARKit。
- Xcode 控制台是否有 ARSession 错误。

### 12.5 车速不准

车速来自 GPS 和 IMU 的简化融合，可能受 GPS 刷新率和信号影响。

建议：

- 室外使用。
- 保持手机固定。
- 避免遮挡 iPhone 天线区域。

## 13. 安全注意事项

必须遵守：

1. 只在封闭卡丁车场使用。
2. 手机必须固定牢靠，不能手持驾驶。
3. 手机不能遮挡视线。
4. AR 行车线只作辅助参考。
5. 驾驶员必须优先观察真实赛道、车辆和工作人员指令。
6. 如果 AR 线偏移或影响判断，立即停止使用。

## 14. 推荐测试流程

第一次测试建议不要直接高速驾驶。

1. 使用 `SampleTrack.json` 或真实赛道 JSON。
2. 在空旷地方启动 App，确认相机和 HUD 正常。
3. 到赛道起点附近低速移动。
4. 观察 GPS 精度是否低于 ±10 米。
5. 完成校准后低速跑一圈。
6. 检查绿/橙/红分段是否符合预期。
7. 再逐步调整线宽、透明度和亮度。

## AI生成赛道

App 的 `赛道` 页面新增 `AI生成`：

1. 点击底部 `赛道`。
2. 点击左上角 `AI生成`。
3. 选择赛道俯视照片。
4. 点击照片上的起终点/终点位置。
5. 输入 AI 接口 Key、Base URL 和 Model ID。Key 会保存在本机 Keychain；Base URL 和 Model ID 会保存在本机设置。
6. 点击 `AI生成并导入赛道`。
7. AI 返回像素行车线点后，App 会按当前定位原点转换为内部 WGS84 赛道并自动导入。
8. 回到 `赛道` 页面选择新赛道，在地图校准页点选你当前所在位置并设置车头方向。

当前内置接口配置：

- 默认 Base URL：`https://api.tutujin.com/v1`，也可以在 App 的 `AI生成` 页面自行修改
- 默认 Model：`claude-3-5-sonnet-20240620`，也可以在 App 的 `AI生成` 页面自行修改
- API 协议：OpenAI 兼容 `POST /chat/completions`

注意：照片本身通常没有真实地理坐标，AI生成结果会按当前GPS位置作为赛道局部坐标原点进行近似转换。正式使用前应通过“赛道”选择流程完成地图位置和方向校准，并低速验证行车线位置。
